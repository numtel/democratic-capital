Democratic Capital Documentation

Creating a group

Anybody can make a new group on Democratic Capital. Immediately after deploying a new group, your account becomes the first member. When there is one member in a group, that member is an administrator and can immediately perform any actions without waiting for a proposal to pass.

It is during this initial administration period, that the group creator should allow at least one election contract, and one registration contract.

Allowing an election contract

As soon as a second member joins the group, the first member is no longer an administrator (except if they added their account address as an allowed contract) so it is very important to create an election contract that can perform administrative transactions on behal of the group.

First the factory for the contract type must be allowed. Click on the 'allowContract' button on your group details page.

Select the factory for the election contract you choose and submit the form.

Election Types

ElectionsSimple

The simplest type of elections which specify the proposal duration, passing threshold, and minimum participation percentage at deployment time.
ElectionsSimpleQuadratic

Like the previous, except votes can include a payment in a specified token that increases the power of the vote by the square root of the payment amount.
ElectionsByMedian

Proposal duration, passing threshold, and minimum participation are set by calculating the median value of member-submitted ballots, giving more democratic power to how the elections proceed.

After allowing the factory, browse to the factory's details page from the link in the 'Allowed Contracts' section of your group details page and click on the button to 'deployNew'.

All election types allow a filter to be set when creating the election contract. This filter provides a list of allowed contracts (and optionally methods too) that can be invoked by proposals created within the contract.

If no filter is specified, the elections contract will accept proposals for any contract and method. In most groups, it is a good idea to have at least one unfiltered elections contract but its duration, passing threshold, and minimum participation values should reflect the immense power of the ability to invoke anything.

For more narrow election contracts filtered to more trivial tasks like setting names and texts, shorter duration, lower passing threshold, and/or lower minimum participation values make sense.

After creating the elections contract, it must be allowed by the group before its successful proposals may invoke their transactions.

Creating a registrations contract

Now that your group has an elections contract, you need a way for other people to join the group. If you're an administator, you can manually add members from the group details page but for all other cases, you'll want to choose one of the registrations contracts to make things simpler.

OpenRegistrations

Allow any account to become a member simply by clicking a button.
RegistrationsByFee

Allow any account to become a member by paying a fee in a specified token.
RegistrationsByElection

Wishful new members submit a proposal under a given election contract with the hope that existing members will approve their membership.

As with the elections contract, you'll first need to allow a factory for one of these registration types, then deploy the contract from the factory details page, then finally allow the new registration contract to invoke on behalf of the group.

It is a also good idea to allow an OpenUnregistrations contract so that members can leave the group if they choose.
